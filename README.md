# Consumer360: Customer Analytics & RFM Segmentation

## Project Overview
This project was developed during my **Data Analytics Internship at Zaalima Development Pvt. Ltd.** It focuses on analyzing retail data from the Olist dataset to provide actionable insights into customer behavior and sales performance.

## Tech Stack
* **Database:** MySQL (Data extraction and transformation)
* **Scripting:** Python (Pandas, SQLAlchemy, Seaborn)
* **IDE:** VS Code
* **Visualization:** Power BI

## Key Features & Analysis
1. **ETL Process:** Cleaned and transformed raw retail data using Python scripts (`etl_process.py`).
2. **RFM Analysis:** Segmented **96.48K customers** into groups like Champions, Loyal, and At Risk based on Recency, Frequency, and Monetary values.
3. **Data Pipeline:** Integrated **MySQL** with **Power BI** for real-time data flow.
4. **Correlation Study:** Used a **Python Seaborn Heatmap** inside Power BI to analyze the relationship between price, freight value, and order volume.

## Business Impact
* **Total Revenue Analyzed:** 13.22M
* **Total Customers:** 96.48K
* **Goal:** To help the marketing team target specific customer segments with personalized campaigns.

### 📂 File Structure
* `Consumer360_Dashboard.pbix`: The interactive Power BI dashboard.
* `rfm_analysis.py`: Python logic for customer segmentation.
* `zaalima_data_queries.sql`: MySQL queries for table creation and data extraction.
* `correlation_analysis.py`: Script for the correlation heatmap visual.
* `Dashboard_Final_Look.png`: Preview of the final dashboard.
